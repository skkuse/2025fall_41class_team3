# InitialView
사용자가 자연어로 질문을 입력하면, LLM이 상황을 이해하고 
정책 추천 결과를 카드 형태로 제공하는 핵심 화면.
  ## 상단 안내 영역 (히어로)
  ### 타이틀
  “[서비스명]이 나의 상황에 꼭 맞는 정책을 찾아드립니다.”
  ### 서브 텍스트
  “자연어로 편하게 질문해보세요. 예: ‘서울에 사는 27살 직장인인데 주거 지원 정책 알려줘.’”

  ---

  ## 채팅 영역
  ### 구성
  - **사용자 메시지 버블**
    - 오른쪽 정렬, 강조 색 배경
    - 최근 발화가 위에서 아래로 순차적으로 쌓이는 구조
  - **시스템(서비스/LLM) 메시지 버블**
    - 왼쪽 정렬, 흰색 또는 연한 배경
    - 메시지 안에 정책 추천 리스트를 카드 형태로 포함 가능
  ### 기능
  - 스크롤 가능한 히스토리
  - 새 질문 입력 시
    a. 사용자 메시지 버블 추가  
    b. “생각 중…” 로딩 버블 표시  
    c. GET /api/policies/recommend 결과를 시스템 메시지 + 정책 카드 리스트로 표시

# ChatView
  # 추천 결과
  LLM 추천 결과는 /api/policies/recommend 응답을 기반으로 카드 형태로 표시.

  ---

  ## 1. 카드 UI
  - 정책명
  - 한 줄 설명 (정책 DB에서 가져올 수 있다면)
  - 추천 이유:
    - “관심 분야(창업)과 일치”
    - “현재 거주 지역과 대상 지역이 일치”
  - 뱃지/태그 리스트 (배경 칩 형태)
  - “자세히 보기” 버튼 또는 카드 전체 클릭 → Policy Detail 페이지로 이동

  ---

  ## 2. 입력 영역
  페이지 하단에 고정된 입력 바

  ### 요소
  - 자연어 입력 필드 (prompt)
    - Placeholder: “지금 상황이나 궁금한 정책을 자유롭게 입력해 주세요.”
  - 전송 버튼 “보내기”

  ### 동작
  전송 클릭 시 → 채팅 영역에 유저 발화 추가 → /api/policies/recommend 호출


# PolicyDetailModal
  # 정책 상세 페이지
  추천 결과 또는 채팅 내 카드에서 선택한 정책에 대한 상세 내용을 제공하는 화면

  ---

  ## 1. 상단 헤더
  - 뒤로가기 버튼: “대화로 돌아가기” 또는 “이전 페이지”
  - (선택) 공유 버튼: URL 복사 / SNS 공유 등

  ---

  ## 2. 요약 카드
  - 정책명
  - (선택) 평점 & 리뷰 개수
  - 정책 분야 태그 (예: 일자리, 창업, 교육지원, 주거 등)
  - 주요 태그 (예: 교육지원 / 보조금 / 프로그램 / 청년 / 저소득층)

  ---

  ## 3. 본문 섹션
  ### 정책 설명
  - 정책의 목적/배경, 제공 내용 개요 등 자유 서술 텍스트

  ### 지원 내용
  - 지원 대상 (연령, 소득 기준, 지역 등)
  - 지원 범위 (금액, 서비스 내용)
  - 기타 조건 (우대사항, 제한사항 등)

  ### 신청 방법
  - 온라인/오프라인 신청 절차
  - 필요 서류
  - 방문 주소, 연락처(전화번호 등)

  ### 신청 기간
  - 형식: YYYY.MM.DD ~ YYYY.MM.DD

  ### 사업 기간 (있을 경우)
  - 정책 시행 기간 또는 프로그램 운영 기간

  ### 신청 링크
  - 외부 신청 페이지 URL / 버튼 라벨: “신청 페이지로 이동”
